## 1. 產品概述

| 欄位 | 內容 |
|------|------|
| 產品名稱 | ShoesReborn 洗鞋店管理系統 |
| 版本 | v1.0 (MVP) |
| 文件版本 | 1.2.0 |
| 建立日期 | 2026-03-04 |

### 摘要

ShoesReborn 是一套專為洗鞋 / 護理店設計的門市營運管理系統。透過數位化開單、鞋物位置追蹤、急件優先管理、會員制度與財務報表五大核心模組，取代現行的紙本收據與 Excel 流程，提升門市作業效率、降低取件錯誤率，並累積顧客消費數據以支援業務決策。

## 2. 背景與問題陳述

### 2.1 現況

| 流程 | 現行做法 |
|------|---------|
| 開單 | 手寫紙本收據（姓名、電話、服務項目、價格、預計取件日） |
| 會員建立 | 第一次來店紙本填寫，無後續追蹤 |
| 鞋物存放 | 儲藏室 + 店內各角落，無系統性標記 |
| 急件管理 | 靠人工記憶或收據排序 |
| 財務記錄 | Excel + Google Forms，手動彙整 |

### 2.2 核心痛點

| 編號 | 痛點 | 影響 |
|------|------|------|
| P-01 | 鞋物散落各處，取件時四處尋找 | 作業效率低，顧客等待時間長，服務體驗差 |
| P-02 | 急件無優先順序管理 | 急件可能延誤，引發顧客投訴 |
| P-03 | 無數位會員資料 | 無法識別回頭客、無法提供個人化服務 |
| P-04 | 無消費行為分析 | 無法識別高價值顧客、無法掌握服務需求趨勢 |
| P-05 | 財務仰賴 Excel 手動作業 | 數據不即時、易出錯、無法快速決策 |

### 2.3 機會

- 洗鞋護理市場需求持續成長，門市數位化可建立差異化競爭優勢
- 會員資料數位化後，可延伸開展行銷活動（生日優惠、集點卡）
- 即時財務報表可協助老闆掌握最佳定價策略與淡旺季規律

## 4. 產品目標與 KPI

### 4.1 業務目標

| 目標 | 說明 |
|------|------|
| 提升取件效率 | 縮短取件作業時間 |
| 降低急件延誤率 | 確保急件準時完成 |
| 建立顧客資產 | 累積可分析的消費資料 |
| 財務透明化 | 即時掌握營收狀況 |

### 4.2 KPI 指標

| KPI | 基準（現況） | 目標（上線 3 個月後） | 衡量方式 |
|-----|------------|---------------------|---------|
| 平均取件等待時間 | 約 10 分鐘（估） | 降低至 3 分鐘以內 | 系統記錄取件開始至完成時間 |
| 急件準時完成率 | 無追蹤 | 達到 95% 以上 | 系統比對急件截止日與完成日 |
| 會員資料建檔率 | 約 20%（估） | 新客 100% 建檔 | 有訂單無會員資料的比率 |
| 財務報表產出時間 | 每月手動數小時 | 即時自動產出 | 系統可用性 |
| 員工開單時間 | 約 5 分鐘（紙本） | 縮短至 2 分鐘以內 | 訂單建立時間戳 |

## 5. 功能需求

### 5.1 模組總覽

```
ShoesReborn 系統
├── 開單管理      ←→ 鞋物位置追蹤
│                ←→ 急件管理
├── 會員管理      → 開單管理
├── 財務報表
└── 系統設定（服務項目、急件費率、位置管理）
```

---

### 5.2 模組一：開單管理

#### User Stories

**US-ORDER-01：建立訂單**

> 身為員工，我希望能快速建立數位訂單，以便取代手寫紙本收據，加速開單流程並減少人為錯誤。

**US-ORDER-02：查詢訂單**

> 身為員工，我希望能透過姓名、電話或訂單編號快速查詢訂單，以便在顧客取件時迅速找到對應記錄。

**US-ORDER-03：更新訂單狀態**

> 身為員工，我希望能更新訂單的處理進度，以便讓老闆與其他員工知道每筆訂單的目前狀態。

**US-ORDER-04：完成取件**

> 身為員工，我希望在顧客取件時能快速確認並完成訂單，以便記錄實際取件時間與最終收費。

#### 訂單狀態機

```
建立訂單         開始作業           處理完成            顧客取件
   │               │                  │                  │
[PENDING] ──→ [IN_PROGRESS] ──→ [READY] ──→ [PICKED_UP]
   │               │
   └──────────────→ [CANCELLED]（老闆授權）
```

| 狀態 | 代碼 | 說明 |
|------|------|------|
| 待處理 | `PENDING` | 訂單建立，尚未開始作業 |
| 處理中 | `IN_PROGRESS` | 已開始洗鞋 / 護理作業 |
| 待取件 | `READY` | 作業完成，等待顧客取件 |
| 已取件 | `PICKED_UP` | 顧客已完成取件 |
| 已取消 | `CANCELLED` | 訂單取消 |

#### 服務項目清單（預設，可由老闆維護）

| 服務代碼 | 服務名稱 | 說明 |
|---------|---------|------|
| SVC-WASH | 洗鞋 | 基本清潔服務 |
| SVC-COATING | 鍍膜 | 防污保護鍍膜 |
| SVC-BAG | 洗包 | 包包清潔服務 |
| SVC-RECOLOR | 補色 | 鞋面補色修復 |

---

### 5.3 模組二：鞋物位置追蹤

**US-LOC-01：指派存放位置**

> 身為員工，我希望在接收鞋物時能指定存放位置，以便任何員工在取件時都能快速定位物品。

**US-LOC-02：查詢物品位置**

> 身為員工，我希望在顧客取件時，系統能直接顯示該訂單鞋物的存放位置，以便不需要四處尋找，縮短取件時間。

**US-LOC-03：管理位置設定**

> 身為老闆，我希望能自訂門市的存放位置標籤（如：A 區層架-1、儲藏室-B），以便系統符合門市實際空間配置。

**US-LOC-04：變更存放位置**

> 身為員工，我希望能在物品移動後更新存放位置，以便位置資訊保持準確。

#### 位置標記規格

- 位置格式：`[區域代碼]-[層架號]-[格位號]`，例如：`A-3-2`（A 區第 3 層架第 2 格）
- 老闆可自行定義區域代碼與最大層架 / 格位數
- 每個格位可存放多件物品
- 一筆訂單支援多個存放位置（如左鞋與右鞋存於不同格位）

---

### 5.4 模組三：急件管理

**US-URGENT-01：標記急件**

> 身為員工，我希望在建立訂單時能標記急件並設定完成截止時間，以便系統能自動計算加急費用並列入優先處理佇列。

**US-URGENT-02：查看急件儀表板**

> 身為老闆或員工，我希望能看到依截止時間排序的急件清單，以便隨時掌握哪些急件需要優先處理。

**US-URGENT-03：急件即將超時警示**

> 身為員工，我希望急件截止時間前 2 小時能收到系統提示，以便及時安排人力確保急件準時完成。

#### 急件規則

| 規則 | 說明 |
|------|------|
| 截止時間 | 急件訂單必填，精確到日 |
| 加急費率 | 依服務項目各自設定；未設定時使用系統全域急件費率（預設 50%） |
| 急件費計算 | 開單時逐項計算：`每項急件費 = 服務小計 × 該服務 urgentFeeRate（或全域費率）`；加總為訂單急件費 |
| 優先排序 | 急件列表依截止日升冪排列，截止日相同者依建立時間排列 |
| 警示條件 | 距截止時間不足 2 小時且狀態非「待取件」或「已取件」 |
| 費率變更 | 修改任何費率不影響歷史已建立訂單的金額 |

---

### 5.5 模組四：會員管理

**US-MEMBER-01：建立會員**

> 身為員工，我希望在新客首次到店時快速建立會員資料，以便系統能記錄該顧客的消費歷史。

**US-MEMBER-02：查詢會員資料**

> 身為員工，我希望輸入電話號碼就能快速找到會員資料與歷史訂單，以便提供個人化服務並減少重複詢問。

**US-MEMBER-03：查看消費摘要**

> 身為老闆，我希望能查看每位會員的累積消費金額與消費次數，以便識別高價值顧客並給予相應優惠。

**US-MEMBER-04：備註特殊需求**

> 身為員工，我希望能在會員資料上記錄特殊偏好或注意事項，以便提供一致且貼心的服務。

#### 會員等級設計（初版，可由老闆調整門檻）

| 等級 | 代碼 | 累積消費門檻 | 說明 |
|------|------|------------|------|
| 一般會員 | `STANDARD` | 0 元以上 | 所有建檔顧客 |
| 銀卡會員 | `SILVER` | 3,000 元以上 | 享有優先排程 |
| 金卡會員 | `GOLD` | 10,000 元以上 | 享有折扣與優先排程 |

---

### 5.6 模組五：財務報表

**US-REPORT-01：查看每日營收摘要**

> 身為老闆，我希望能查看每日的訂單數量與總營收，以便快速掌握門市當日業績。

**US-REPORT-02：查看月度財務報表**

> 身為老闆，我希望能查看月度的營收趨勢、服務項目分佈，以便了解哪種服務最受歡迎、哪段時間是旺季。

**US-REPORT-03：查看服務項目收益分析**

> 身為老闆，我希望能按服務類型分析營收佔比，以便決策是否調整服務定價或推廣特定服務。

**US-REPORT-04：匯出報表**

> 身為老闆，我希望能將報表匯出為 CSV 格式，以便在 Excel 進行進一步的自訂分析。

## 7. 使用者旅程

### 7.1 開單流程

```
顧客進店
   ↓
是否為會員？
   ├── 否 → 員工建立會員資料（姓名 / 電話 / 備註）
   └── 是 → 員工輸入電話查詢，自動帶入會員資料
   ↓
選擇服務項目（洗鞋 / 鍍膜 / 洗包 / 補色）
   ↓
填寫鞋物資訊（品牌 / 款式 / 數量 / 狀況備註）
   ↓
是否為急件？
   ├── 是 → 設定截止日期，系統自動加計急件費
   └── 否 → 設定一般預計取件日
   ↓
確認總金額
   ↓
系統建立訂單（產生訂單編號 SR-YYYYMMDD-XXXX）
   ↓
員工指定存放位置（如：A-3-2）
   ↓
顧客收到訂單編號 / 告知取件日
```

### 7.2 取件流程

```
顧客到店取件
   ↓
顧客提供訂單編號 或 姓名 + 電話
   ↓
系統顯示訂單詳情（含鞋物存放位置）
   ↓
員工依位置指示前往取回鞋物
   ↓
顧客確認物品正確
   ↓
員工點擊「完成取件」
   ↓
系統更新訂單狀態為 PICKED_UP
系統記錄實際取件時間
會員累積消費金額自動更新
```

### 7.3 急件管理流程

```
急件訂單建立
   ↓
系統加入急件佇列（依截止日排序）
   ↓
距截止時間 < 2 小時 且 狀態非 READY？
   └── 是 → 急件儀表板標紅警示
           員工確認並優先處理
   ↓
處理完成 → 更新狀態為 READY → 等待顧客取件
```

## 8. 驗收條件

### 8.1 開單管理模組

- [ ] 員工可在 2 分鐘內完成一筆新訂單的建立
- [ ] 訂單編號格式符合 `SR-YYYYMMDD-XXXX`，當日流水號不重複
- [ ] 新客必須先建立會員才能建立訂單；舊客可透過電話自動帶入資料
- [ ] 勾選急件時，截止日期欄位變為必填，且系統自動計算急件費用
- [ ] 訂單建立後，狀態為 `PENDING`
- [ ] 訂單狀態只能依照合法轉換路徑流轉
- [ ] 已完成取件（PICKED_UP）的訂單不可再被修改
- [ ] 刪除訂單需老闆角色，操作留存 Audit Log
- [ ] 搜尋訂單可透過訂單編號、顧客姓名、顧客電話其中任一查詢
- [ ] 列表支援依狀態、是否急件、建立日期區間篩選

### 8.2 鞋物位置追蹤模組

- [ ] 每筆訂單可指定一個或多個位置代碼
- [ ] 查詢訂單詳情時，清楚顯示所有存放位置的完整描述與備註
- [ ] 員工可隨時更新位置，更新後立即生效
- [ ] 老闆可新增 / 停用位置標籤，停用後不可被新訂單選擇
- [ ] 停用已有訂單存放其中的位置時，系統警示但不強制阻止

### 8.3 急件管理模組

- [ ] 急件訂單在所有列表中有明顯視覺標示（如紅色標籤）
- [ ] 急件列表依截止日升冪排列
- [ ] 距截止時間不足 2 小時且狀態不為 `READY` 或 `PICKED_UP` 的急件，顯示警示標示
- [ ] 急件費計算：優先使用各服務項目的急件費率，未設定者使用系統全域費率
- [ ] 急件費 = 各訂單明細小計 × 對應費率之加總
- [ ] 急件費率變更不影響歷史已建立訂單的金額

### 8.4 會員管理模組

- [ ] 同一手機號碼不可重複建立會員
- [ ] 顧客完成取件後，系統自動更新該會員的累積消費金額與到店次數
- [ ] 系統依累積消費金額自動判斷並更新會員等級
- [ ] 員工可查詢會員基本資料與歷史訂單，但不可修改會員等級
