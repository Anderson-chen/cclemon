# Gemini 開發前檢查清單

在開始修改或新增程式碼前，請參考以下清單，確保開發流程順暢且品質穩定。

---

### 1. 需求與目標 (Requirement & Goal)
- [ ] 我是否已完全理解需求？
- [ ] 我是否已將複雜的需求拆解成數個可管理的小任務？
- [ ] 我是否已思考過可能的邊界情況 (edge cases)？

### 2. 設計與架構 (Design & Architecture)
- [ ] 我的修改是否符合現有的專案架構？
- [ ] 是否有可重用的元件或函式庫可以使用？
- [ ] 這次的修改是否會影響到其他模組？如果會，我該如何處理？

### 3. 編碼風格 (Coding Style)
- [ ] 程式碼風格是否與專案既有風格一致？
- [ ] 命名是否清晰且有意義？
- [ ] 是否在需要的地方加上了註解？

### 4. 測試 (Testing)
- [ ] 我是否為新的功能或修正撰寫了單元測試？
- [ ] 所有相關的測試是否都已通過？
- [ ] 我是否手動測試了主要的使用流程？

### 5. 提交與審查 (Commit & Review)
- [ ] Git commit 訊息是否清晰明瞭？
- [ ] 我是否只提交了與本次任務相關的程式碼？
- [ ] 我是否準備好向團隊成員解釋我的修改？
